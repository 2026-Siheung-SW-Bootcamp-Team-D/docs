# 외부 API 계약 PoC v2

여러 참여자의 출발지를 입력하면 각자의 대중교통 도달권을 겹치고, 공통 영역 안의 후보를 실제 이동시간으로 비교하는 실행형 PoC입니다.

단순히 “중간 지점을 검색한다”가 아니라 다음 흐름을 한 화면에서 확인할 수 있습니다.

1. 참여자별 실제 장소를 Kakao로 검색하고 선택합니다.
2. ODsay에서 각 참여자의 대중교통 도달권을 구합니다.
3. 모두가 갈 수 있는 공통 영역을 계산합니다.
4. 공통 영역 안의 역·터미널·공공장소 후보를 축적합니다.
5. TMAP 대중교통 경로로 모든 참여자의 실제 이동시간을 비교합니다.
6. 도달 불가 인원, 최장 이동시간, 평균 이동시간 순으로 설명 가능한 후보를 보여줍니다.

## 5분 안에 실행하고 접속하기

### 1. 실행 전 확인

다음 항목이 준비되어 있어야 합니다.

- Node.js 18 이상
- `backend/.env` 파일
- 아래 외부 API 키 3개
  - `KAKAO_REST_API_KEY`
  - `ODSAY_API_KEY`
  - `TMAP_APP_KEY`

프로젝트 폴더는 다음처럼 `backend`와 `docs`가 나란히 있는 구조를 기준으로 합니다.

```text
2026-Siheung-SW-Bootcamp-Team-D/
├── backend/
│   └── .env
└── docs/
    └── api-validation/
```

### 2. 서버 실행

터미널을 열고 **전체 프로젝트 루트**에서 아래 명령을 그대로 실행합니다.

```bash
cd docs/api-validation

# backend/.env의 API 키를 현재 터미널에 불러옵니다.
set -a
source ../../backend/.env
set +a

# 최초 한 번만 의존성을 설치합니다.
npm install

# 구현이 정상인지 확인합니다.
npm test

# 시각화 서버를 실행합니다.
npm run serve
```

정상적으로 실행되면 터미널에 다음 주소가 표시됩니다.

```text
http://localhost:4173
```

### 3. 브라우저 접속

서버를 실행한 터미널은 그대로 둔 상태에서 Chrome, Safari, Edge 등의 브라우저로 아래 주소를 엽니다.

➡️ **[http://localhost:4173 접속하기](http://localhost:4173)**

`localhost`는 현재 컴퓨터를 뜻합니다. 인터넷에 배포된 주소가 아니므로 반드시 서버를 실행한 컴퓨터에서 접속해야 합니다.

### 4. 서버 종료

체험을 마친 뒤 서버를 실행한 터미널에서 `Ctrl+C`를 누르면 종료됩니다.

API 키는 서버 프로세스에서만 사용하며 HTML, 브라우저 응답, 결과 보고서에는 포함하지 않습니다.

## 화면에서 체험하는 순서

1. 참여자 1의 출발지를 검색하고 결과 하나를 선택합니다.
2. 참여자 2도 같은 방식으로 선택합니다.
3. 필요하면 `참여자 추가`로 최대 5명까지 늘립니다.
4. 30분, 45분, 60분 중 도달시간을 선택합니다.
5. `지역 찾기 실행`을 누릅니다.
6. 진행 단계가 끝나면 지도, 후보 순위, 참여자별 이동시간 표를 확인합니다.
7. 미리 준비된 시나리오 탭과 직접 만든 시나리오를 오가며 결과를 비교합니다.

외부 API 호출은 보통 수 초에서 수십 초가 걸립니다. 진행 중에는 현재 처리 단계와 호출 수를 화면에 표시합니다.

## 접속되지 않을 때

### `localhost:4173`에 연결할 수 없음

- `npm run serve`를 실행한 터미널이 아직 열려 있는지 확인합니다.
- 터미널에 오류가 표시됐다면 오류 내용을 먼저 확인합니다.
- 이미 다른 프로그램이 4173 포트를 사용 중인지 확인합니다.

### API 키가 없다는 오류

프로젝트 루트에서 아래 파일이 존재하는지 확인합니다.

```text
backend/.env
```

그리고 `.env`에 다음 이름이 정확히 들어 있는지 확인합니다. 실제 키 값은 README나 Git에 올리지 않습니다.

```dotenv
KAKAO_REST_API_KEY=발급받은_키
ODSAY_API_KEY=발급받은_키
TMAP_APP_KEY=발급받은_키
```

`.env`를 수정했다면 실행 중인 서버를 `Ctrl+C`로 종료한 뒤 환경변수 로딩과 `npm run serve`를 다시 실행합니다.

### 장소 검색이나 계산이 오래 걸림

실제 Kakao·ODsay·TMAP API를 호출하므로 네트워크 상태나 공급자 응답 속도에 따라 수십 초가 걸릴 수 있습니다. 화면의 진행 문구가 바뀌고 있다면 기다려 주세요.

### 화면은 열리지만 지도가 보이지 않음

지도 타일은 OpenStreetMap에서 불러옵니다. 인터넷 연결을 확인하고 페이지를 새로고침합니다.

## 실 API 시나리오 보고서 다시 만들기

```bash
npm run poc
```

결과는 `poc-v2/output/report-data.json`에 저장됩니다. API 키와 인증 헤더는 저장하지 않습니다.

## 판정 기준

- `SUCCEEDED`: 공통 영역이 있고, 실제 이동시간 비교까지 완료
- `NO_INTERSECTION`: 모든 참여자가 함께 도달할 수 있는 영역이 없음
- `FAILED`: 외부 API 오류, 응답 계약 불일치, 입력 오류 등으로 계산 실패

`NO_INTERSECTION`은 실패가 아니라 제품이 사용자에게 설명해야 하는 정상적인 결과입니다.

## 구현 범위와 한계

- 로컬 PoC용 인메모리 작업 큐이므로 서버를 재시작하면 직접 실행한 작업 기록은 사라집니다.
- 후보 검색 결과에는 같은 역의 노선별 항목이나 같은 건물의 다른 장소가 함께 나올 수 있습니다. 본 서비스에서는 좌표·주소·장소 유형을 이용한 후보 병합 규칙이 필요합니다.
- 외부 API 요금, 호출 제한, 약관은 실제 서비스 적용 전에 각 공급자 콘솔에서 다시 확인해야 합니다.
- 현재 PoC는 기능·응답 계약 검증 목적이며 다수 사용자의 동시 부하 시험은 포함하지 않습니다.
