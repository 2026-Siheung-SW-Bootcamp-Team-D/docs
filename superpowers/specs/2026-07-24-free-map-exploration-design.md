# 자유 지도 탐색 PoC 설계

## 목표

출발지 기반 계산은 사용자가 약속 지역의 감을 잡도록 돕는 참고 기능으로 제한한다. 사용자는 공통 도달 영역과 관계없이 카카오맵의 임의 위치를 선택하고, 그 주변 장소를 검색해 공동 후보에 추가할 수 있다.

## 결정

- 환승역 중복 병합은 이번 범위에서 제외한다.
- 도달시간은 후보를 제외하거나 순위를 결정하지 않는다.
- 설정 시간은 ODsay 공통 도달 영역을 그리는 입력으로만 사용한다.
- 자동 지역 후보는 출발지들의 대표 중심에 가까운 교통 거점 최대 3개만 제시한다.
- 자동 지역 후보의 TMAP 이동시간은 설명용 정보이며 시간 초과를 허용한다.
- 빨간 공통 영역은 추천 가능 범위가 아닌 참고 레이어다.
- 사용자는 빨간 영역 안팎을 구분하지 않고 지도 어디든 탐색 핀을 놓을 수 있다.
- 탐색 핀 주변을 카테고리 또는 키워드로 검색할 수 있다.
- 검색 결과는 기존 공동 후보 목록에 추가할 수 있다.
- 카카오맵 Web SDK를 사용하고 ngrok의 고정 개발 도메인으로 브라우저에서 검증한다.

## 최소 사용자 흐름

1. 참여자 출발지를 입력하고 도달시간을 선택한다.
2. PoC가 참여자별 도달권과 공통 영역을 카카오맵에 표시한다.
3. PoC가 출발지 대표 중심 근처의 자동 탐색점 최대 3개를 표시한다.
4. 사용자는 자동 탐색점을 선택하거나 지도 임의 위치를 클릭한다.
5. 클릭 위치에 탐색 핀이 표시된다.
6. 사용자는 음식점, 카페, 문화시설, 관광명소 또는 직접 키워드로 주변 장소를 검색한다.
7. 검색 결과를 공동 후보에 추가한다.
8. 필요할 때만 공동 후보의 이동시간을 다시 계산한다.

## 산정 규칙

### 자동 탐색점

- 참여자 좌표의 평균을 대표 중심으로 사용한다.
- 기존 Kakao 교통 거점 검색 결과 중 대표 중심과 가까운 순서로 최대 3개를 선택한다.
- ODsay 공통 영역 포함 여부와 TMAP 이동시간은 정렬 조건으로 사용하지 않는다.
- 구현 복잡도를 줄이기 위해 상권 밀도 점수와 환승역 클러스터링은 도입하지 않는다.

### 자유 탐색점

- 지도 클릭 좌표를 그대로 사용한다.
- 공통 영역 밖이어도 허용한다.
- 마지막으로 클릭한 탐색점 하나만 유지한다.
- 검색 반경은 기존 기본값 1,000m를 유지한다.
- 서버는 유효한 위도·경도, 검색 반경, 검색어 또는 카테고리만 검증한다.

## 구성

### 브라우저

- 카카오맵 지도, 도달권, 공통 영역, 자동 후보 및 탐색 핀을 표시한다.
- 지도 클릭 좌표를 장소 검색 요청에 전달한다.
- JavaScript 키는 브라우저 지도 SDK 로딩에만 사용한다.

### PoC 서버

- 기존 카카오 REST API 키를 사용해 장소를 검색한다.
- 장소 검색은 기존 `hubId` 또는 자유 좌표 `lon`, `lat` 중 하나를 받는다.
- 기존 공동 후보 저장과 온디맨드 이동시간 계산을 재사용한다.

## 오류 처리

- 카카오맵 JavaScript 키가 없으면 설정 방법을 화면에 표시한다.
- 자유 좌표가 유효하지 않으면 장소 검색 요청을 거부한다.
- 카카오 장소 검색 결과가 없으면 다른 위치, 반경 또는 검색어를 안내한다.
- ODsay 공통 영역이 없더라도 지도 자유 탐색은 계속 사용할 수 있다.

## ngrok 검증

- 로컬 PoC 서버를 실행한다.
- ngrok 무료 계정에 할당된 고정 개발 도메인으로 PoC 포트를 공개한다.
- 해당 HTTPS 도메인을 카카오 JavaScript SDK 허용 도메인에 등록한다.
- README에 로컬 실행, ngrok 실행, 카카오 도메인 등록, 접속 순서를 기록한다.

## 완료 기준

- 자동 후보가 도달시간 공정성 대신 대표 중심과의 거리로 결정된다.
- 설정 시간을 초과한 후보도 결과에 남는다.
- 카카오맵에 도달권과 공통 영역이 표시된다.
- 공통 영역 밖을 클릭해도 탐색 핀이 생성된다.
- 탐색 핀 주변 카테고리 및 키워드 검색이 동작한다.
- 검색 결과를 공동 후보에 추가할 수 있다.
- 기존 공동 후보 온디맨드 이동시간 계산이 유지된다.
- 자동 테스트와 ngrok 브라우저 검증 절차가 README에 남는다.
